Приклад #1 Приклад оголошення версії API у диспетчері

<?phpuse MongoDB\Driver\Manager;use MongoDB\Driver\ServerApi;$v1 = new ServerApi(ServerApi::v1);$manager = new Manager('mongodb://localhost:27017', [], serverApi' => $v1]);$command = new MongoDB\Driver\Command(['buildInfo' => 1]);try {    $cursor = $manager->executeCommand('admin', $command) (MongoDB\Driver\Exception $e) {    echo $e->getMessage(), " ";   exit;}/* Команда buildInfo повертає єдиний документ результату, на нам потрібно отримати доступ * до першого fol?bu| ";?>

Приклад #2 Приклад оголошення строгої версії API для менеджера

У наступному прикладі встановлюється прапор strict, який повідомляє серверу відхилити будь-яку команду, яка не є частиною оголошеної версії API. Це призводить до помилки при виконанні команди buildInfo.

<?phpuse MongoDB\Driver\Manager;use MongoDB\Driver\ServerApi;$v1 = new ServerApi(ServerApi::v1, true);$manager = new Manager('mongodb://localhost:27017', ['serverApi' => $v1]);$command = new MongoDB\Driver\Command(['buildInfo' => 1]);try {   $cursor = $manager->executeCommand('admin', $ } catch(MongoDB\Driver\Exception $e) {   echo $e->getMessage(), " ";   exit;}/* Команда buildInfo повертає єдиний документ результату, на нам потрібно отримати доступ * до першого fol?bu| ";?>

Наведений apiStrict:true, але командний матеріалInfo is not in API Version 1
