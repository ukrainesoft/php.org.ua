#pcntl_wait

(PHP 5, PHP 7, PHP 8)

pcntl_wait — Очікує або повертає статус породженого дочірнього
процесу

### Опис

**pcntl_wait**(int `&$status`, int `$flags` u003d 0, array
`&$resource_usage` u003d \[\]): int

Функція очікування призупиняє виконання поточного процесу
доки дочірній процес не завершиться або поки не буде отримано
сигнал, який завершує поточний процес або викликає функцію обробки
сигналу. Якщо дочірній процес вже завершився на час виклику (так
звані "зомбі" процеси), функція негайно поверне управління.
Зверніться до вашого системного посібника (man) wait(2) для уточнення
специфіки роботи wait у вашій системі.

> **Примітка**:
>
> Ця функція еквівалентна виклику
> [pcntl_waitpid()](function.pcntl-waitpid.md) з аргументом
> `process_id` рівним `-1` і без аргументу `flags`.

### Список параметрів

`status`
**pcntl_wait()** розмістить інформацію про статус за посиланням в аргументі
`status`, який може бути переданий у такі функції:
[pcntl_wifexited()](function.pcntl-wifexited.md),
[pcntl_wifstopped()](function.pcntl-wifstopped.md),
[pcntl_wifsignaled()](function.pcntl-wifsignaled.md),
[pcntl_wexitstatus()](function.pcntl-wexitstatus.md),
[pcntl_wtermsig()](function.pcntl-wtermsig.md) та
[pcntl_wstopsig()](function.pcntl-wstopsig.md).

`flags`
Якщо системний виклик wait3 (man 2 wait3) доступний у вашій системі (як
правило в BSD-подібних системах), ви можете задати необов'язковий
аргумент `flags`. Якщо цей параметр не заданий, буде використано
стандартний системний дзвінок wait. Якщо системний виклик wait3 недоступний
для вашої системи, передача значення аргументу `flags` не матиме
значення. Значення аргументу `flags` - це бітова маска, яка може
приймати значення нуль або більше шляхом логічного об'єднання
наступних констант:

| | |
|-------------|--------------------------------------- -------------------------------------------------- -------|
| `WNOHANG` | Негайно повернути управління, якщо жоден із дочірніх процесів не завершено |
| `WUNTRACED` | Повернути управління для зупинених дочірніх процесів, статус яких ще не повідомлено |

**Можливі значення для `flags`**

### Значення, що повертаються

**pcntl_wait()** повертає ID завершеного дочірнього процесу, -1 в
у разі виникнення помилки або нуль, якщо `WNOHANG` був переданий у
аргумент `options` (на системах з доступним системним викликом wait3) та
був доступних дочірніх процесів.
