mysqli::close

mysqli_close

(PHP 5, PHP 7, PHP 8)

mysqli::close -- mysqli_close — Закриває раніше відкрите з'єднання з базою даних

Опис

Об'єктно-орієнтований стиль

public mysqli::close(): bool

Процедурний стиль

mysqli_close(mysqli $mysql): bool

Закриває відкрите з'єднання з базою даних.

Відкриті непостійні з'єднання MySQL та набори результатів автоматично закриваються під час знищення їх об'єктів. Явне закриття відкритих з'єднань та звільнення наборів результатів не обов'язково. Однак рекомендується закрити з'єднання, як тільки скрипт завершить виконання всіх своїх операцій з базою даних, якщо йому ще належить велика обробка після отримання результатів.

Список параметрів

mysql Тільки для процедурного стилю: об'єкт mysqli, отриманий за допомогою mysqli_connect() або mysqli_init().

Значення, що повертаються

Повертає true у разі успішного виконання або false у у разі виникнення помилки.

Примітки

Примітка:

mysqli_close() не закриває постійні з'єднання. Для отримання подробиці дивіться посібник з persistent > connections.

Дивіться також

  • mysqli::__construct() - Встановлює нове з'єднання з сервером MySQL
  • mysqli_init() - Ініціалізує MySQLi та повертає об'єкт для використання у функції mysqli_real_connect()
  • mysqli_real_connect() - Встановлює з'єднання з сервером mysql
  • mysqli_free_result() - Звільняє пам'ять, зайняту результатами запиту
