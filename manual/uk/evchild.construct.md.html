EvChild::__construct

(PECL ev >= 0.2.0)

EvChild::__construct — Створює об'єкт спостерігача EvChild

Опис

public EvChild::__construct( int $pid, bool $trace, callable $callback ,

mixed $data = null , int $priority = 0 )

Створює об'єкт спостерігача EvChild.

Викликає callback-функцію, коли настала подія зміни статусу процесу з ідентифікатором pid (або будь-яким PID, якщо pid заданий як 0). Статус процесу змінюється, коли процес завершується, або коли його вбивають, або, якщо trace дорівнює true, коли він зупинено або відновлено. Іншими словами, коли процес отримує сигнал SIGCHLD , Ev отримує статус exit/wait для всіх змінених/зомбі дочірніх процесів і викликає callback-функцію.

Правильно встановлювати дочірнього спостерігача після того, як EvChild завершився, але до запуску наступної ітерації подієвого циклу. Наприклад, спочатку викликається fork, після чого новий дочірній процес може вийти, і лише після цього у батьку встановлюється спостерігач EvChild для нового PID .

Ви можете отримати доступ до статусів exit/tracing та pid використовуючи властивості об'єкта спостерігача rstatus та rpid.

Кількість PID-спостерігачів для кожного PID не обмежена. Всі вони будуть викликані.

Метод EvChild::createStopped() не стартує(не активує) створеного спостерігача.

Список параметрів

pid Очікує зміни статусу процесу з ідентифікатором PID (або будь-якого процесу, якщо PID заданий як 0).

trace Якщо false, то активація спостерігача відбувається лише за завершення процесу. Якщо true, то активація відбувається також за зупинки/відновлення процесу.

callback Дивіться Callback-функції спостерігачів.

data Довільні дані, пов'язані зі спостерігачем.

priority Пріоритет спостерігача
