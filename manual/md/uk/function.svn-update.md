# svn_update

(PECL svn \>u003d 0.1.0)

svn_update — Оновлює робочу копію

### Опис

**svn_update**(string `$path`, int `$revno` u003d SVN_REVISION_HEAD, bool
`$recurse` u003d **`true`**): int

Оновлює робочу копію, розташовану в `path` до ревізії `revno`. Якщо
параметр `recurse` встановлено в **`true`**, директорії будуть оновлені
рекурсивно.

### Список параметрів

`path`
Шлях до локальної робочої копії.

> **Примітка**: Відносні шляхи будуть обчислені, наче поточна
> робоча директорія була домашньою папкою самого PHP. Щоб
> використовувати робочу директорію скрипта, що викликає, використовуйте
> [realpath()](function.realpath.md) або dirname(\_\_FILE\_\_).

`revno`
Номер ревізії для оновлення за промовчанням **`SVN_REVISION_HEAD`**.

`recurse`
Чи слід рекурсивно оновлювати директорії.

### Значення, що повертаються

У разі успішного виконання повертає новий номер ревізії або
**`false`** у разі виникнення помилки.

### Приклади

**Приклад #1 Базовий приклад**

Нижченаведений приклад ілюструє базове застосування цієї функції:

`<?phpecho svn_update(realpath('working-copy'));?> `

Результатом виконання цього прикладу буде щось подібне:

**Увага**

Ця функція є ЕКСПЕРИМЕНТАЛЬНОЮ. Поведінка цієї функції, її ім'я

PHP без попередження. Використовуйте цю функцію на свій страх та ризик.
