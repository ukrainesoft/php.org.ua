#imap_reopen

(PHP 4, PHP 5, PHP 7, PHP 8)

imap_reopen — Відкриває потік IMAP до нової скриньки

Опис

imap_reopen( IMAP\Connection $imap, string $mailbox, int $flags = 0, int $retries = 0 ): bool

Перевідкриває вказаний потік до ящика mailbox на сервері IMAP або NNTP.

Список параметрів

imap Примірник IMAP\Connection.

mailbox Ім'я поштової скриньки. Докладніше читайте у розділі про функцію imap_open()

Увага Якщо imap.enable_insecure_rsh не вимкнено, то передача в цей параметр не перевірених даних не безпечна.

flags flags - бітова маска з однієї або кількох констант:

  • OP_READONLY - відкрити поштову скриньку тільки для читання
  • OP_ANONYMOUS - не використовувати та не оновлювати .newsrc для новин (тільки NNTP)
  • OP_HALFOPEN - відкрити з'єднання, але не підключатися до поштову скриньку для ім'я IMAP та NNTP.
  • OP_EXPUNGE - мовчки виконати видалення помічених для видалення повідомлень у потоці
  • CL_EXPUNGE - автоматично видаляти всі позначені для видалення повідомлення при закритті поштової скриньки (див. imap_delete() та imap_expunge())

retries Максимальна кількість спроб з'єднання

Значення, що повертаються

Повертає true, якщо потік перевідкритий і false, якщо ні.
