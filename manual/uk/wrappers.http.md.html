http:// -- https:// — Доступ до URL-адрес за протоколом HTTP(s)

Опис

Надає доступ лише для читання файлів/ресурсів через HTTP. за використовується HTTP 1.0 GET. Для підтримки віртуальних хостів на На основі імен разом із запитом надсилається заголовок Host:. Якщо ви сконфігурували рядок user_agent, використовуючи ваш файл php.ini або контекст потоку, то вона також буде включена до запит.

Цей потік також дозволяє отримати доступ до вмісту ресурсу; заголовки зберігаються у змінній $http_response_header.

Якщо важливо знати URL, з якого було отримано документ (після всіх переадресацій, які були зроблені), то вам необхідно обробити серію заголовків відповідей, що повертаються потоком.

INI-директива from буде використовуватись для заголовка From:, якщо встановлена і не перевизначена в контексті Контекстні опції та параметри.

Використання

  • http://example.com
  • http://example.com/file.php?var1=val1&var2=val2
  • http://user:password@example.com
  • https://example.com
  • https://example.com/file.php?var1=val1&var2=val2
  • https://user:password@example.com

Примітки

Примітка: Протокол HTTPS підтримується лише коли модуль openssl увімкнено.

З'єднання HTTP призначені лише для читання; запис даних або копіювання файлів у HTTP-ресурс не підтримується.

Надсилання запитів POST і PUT, наприклад, може бути виконано з за допомогою HTTP-контексту.
