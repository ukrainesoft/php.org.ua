#curl_share_setopt

(PHP 5 >= 5.5.0, PHP 7, PHP 8)

curl_share_setopt — Встановити опції розділюваного обробника cURL

Опис

curl_share_setopt(CurlShareHandle $share_handle, int $option, mixed $value): bool

Встановлює опції оброблюваного cURL.

Список параметрів

share_handle Обробник, що розділяється, cURL, що повертається curl_share_init().

option

Опція Опис
CURLSHOPT_SHARE Задає тип даних, які потрібно розділяти.
CURLSHOPT_UNSHARE Задає тип даних, які не треба розділяти.

value

Значення Опис
CURL_LOCK_DATA_COOKIE Поділяє дані cookie.
CURL_LOCK_DATA_DNS Поділяє кеш DNS. Зверніть увагу, що якщо ви використовуєте множинний обробник cURL, всі додані обробники за замовчуванням будуть розділяти DNS-кеш.
CURL_LOCK_DATA_SSL_SESSION Розділяє ідентифікатори сесії SSL, скорочуючи час, що витрачається на підтвердження (handshake) SSL при повторному з'єднанні з тим самим сервером. Зверніть увагу, що ідентифікатори сесії SSL будуть за замовчуванням перевикористовуватися тим самим обробником.

Значення, що повертаються

Повертає true у разі успішного виконання або false у у разі виникнення помилки.

Список змін

Версія Опис
8.0.0 share_handle expects a CurlShareHandle instance now; Попередньо, як ресурс був виявлений.

Приклади

Приклад #1 Приклад використання curl_share_setopt()

У цьому прикладі ми створюємо оброблюваний обробник cURL, додаємо в нього два звичайні обробники і запускаємо їх. Вони будуть використовувати одні й ті ж кукі.
