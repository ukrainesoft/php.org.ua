#db2_prepare

(PECL ibm_db2 >= 1.0.0)

db2_prepare — Підготовка SQL-запиту до виконання

Опис

db2_prepare(resource $connection, string $statement, array $options = ?): resource

db2_prepare() створює підготовлений SQL-запит, який може включати 0 або більше маркерів параметрів (символів ?), що становлять вхідні параметри, параметри виводу або вхідні параметри та параметри виведення. Ви можете передати параметри підготовленому запиту за допомогою db2_bind_param() або тільки для вхідних значень у вигляді масиву, переданого в db2_execute().

Використання підготовлених запитів у вашій програмі дає три Основні переваги:

  • Продуктивність: під час використання підготовленого запиту, сервер бази даних створює оптимізований план доступу для вилучення даних за допомогою цього запиту. Наступне виконання підготовленого запиту за допомогою db2_execute() дозволяє операторам повторно використовувати цей план доступу та дозволяє уникнути накладних витрат на динамічне створення нового плану доступу кожного виконуваного вами запиту.

  • Безпека: при використанні підготовленого запиту, ви можете увімкнути маркери параметрів для вхідних значень. Коли ви виконуєте підготовлений запит із вхідними значеннями для заповнювачів, сервер бази даних перевіряє кожне вхідне значення, щоб переконатися, що тип відповідає визначенню стовпця або визначення параметра.

  • Розширений функціонал: Маркери параметрів не лише дозволяють передавати вхідні значення в підготовлені SQL-запити, вони також дозволяють витягувати параметри OUT і INOUT зі збережених процедур з db2_bind_param().

Список параметрів

connection Допустима змінна ресурсу підключення до бази даних, що повертається функцією db2_connect() або db2_pconnect().

statement SQL-запит, який необов'язково містить один або кілька маркерів параметрів.

options Асоціативний масив містить параметри запиту. Параметри можна використовувати для запиту курсора, що прокручується, на серверах баз даних, що підтримують цю функцію.

Опис допустимих опцій запиту дивіться у розділі db2_set_option().

Значення, що повертаються

Повертає ресурс оператора, якщо SQL-запит успішно проаналізовано та підготовлений сервером бази даних. Повертає false, якщо сервер бази даних повернули помилку. Ви можете визначити, яка помилка була повернуто, викликавши db2_stmt_error() або db2_stmt_errormsg().

Дивіться також

  • db2_bind_param() - Зв'язує змінну PHP з параметром SQL-виразу
  • db2_execute() - Виконує підготовлений SQL-запит
  • db2_stmt_error() - Повертає рядок, що містить SQLSTATE, повернутий SQL-оператором
