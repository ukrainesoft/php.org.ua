mysql_field_type

(PHP 4, PHP 5)

mysql_field_type — Повертає тип вказаного поля з результату запиту

Увага

Цей модуль застарів, починаючи з версії PHP 5.5.0, і вилучений до PHP 7.0.0. Використовуйте замість нього MySQLi або PDO_MySQL. Дивіться також інструкцію MySQL: вибір API. Альтернативи для цієї функції:

  • mysqli_fetch_field_direct() [type]
  • PDOStatement::getColumnMeta() [driver:decl_type] або [pdo_type]

Опис

mysql_field_type(resource $result, int $field_offset): string

Функція mysql_field_type() аналогічна функції mysql_field_name(). Аргументи однакові, але замість імені колонки повертається її тип.

Список параметрів

result Оброблюваний результат запита. Цей результат може бути отриманий за допомогою функції mysql_query().

field_offset Числове усунення поля. field_offset починається з 0. Якщо field_offset не існує, генерується помилка рівня E_WARNING.

Значення, що повертаються

Поля можуть бути наступних типів: 'int', 'real', 'string', 'blob' та інших, докладно описаних » документації MySQL.

Примітки

Примітка:

Для зворотної сумісності може бути використаний наступний застарілий псевдонім: mysql_fieldtype()

Дивіться також

  • mysql_field_name() - Повертає назва вказаної колонки результату запиту
  • mysql_field_len() - Повертає довжину вказаного поля
