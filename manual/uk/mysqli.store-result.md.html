mysqli::store_result

mysqli_store_result

(PHP 5, PHP 7, PHP 8)

mysqli::store_result -- mysqli_store_result -- Передає на клієнта результуючий набір останнього запиту

Опис

Об'єктно-орієнтований стиль

public mysqli::store_result(int $mode = 0): mysqli_result|false

Процедурний стиль

mysqli_store_result(mysqli $mysql, int $mode = 0): mysqli_result|false

Передає набір останнього запиту на з'єднанні mysql. Подальша робота з цим набором здійснюється функцією mysqli_data_seek().

Список параметрів

mysql Тільки для процедурного стилю: об'єкт mysqli, отриманий за допомогою mysqli_connect() або mysqli_init().

mode Опція, що встановлюється. Може мати одне з наступних значень:

Ім'я Опис
MYSQLI_STORE_RESULT_COPY_DATA

Valid options

Значення, що повертаються

Повертає буферизований об'єкт результату запиту або false у разі помилки.

Примітка:

mysqli_store_result() повертає false, якщо запит не повертає результуючої таблиці (наприклад, у разі вираження INSERT). Також функція поверне false, якщо дані з результуючого набору не вдалося прочитати. Наявність помилки можна перевірити функцією mysqli_error(), яка в цьому у разі поверне непустий рядок; mysqli_errno() поверне ненульове значення; і mysqli_field_count() також поверне ненульове значення. Також можливою причиною повернення false після успішного виклику mysqli_query() може бути надто великий результуючий набір (бракує пам'яті для його Розміщення). Якщо функція mysqli_field_count() повертає ненульове значення, значить, запит повернув непустий результуючий набір.

Приклади

Дивіться mysqli_multi_query().

Примітки

Примітка:

Навіть попри хорошу практику очищати пам'ять, зайняту результатами запитів за допомогою функції mysqli_free_result(), якщо mysqli_store_result() передає великий результуючий набір, це може стати проблемою.
